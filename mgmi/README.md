# MGMI &mdash; MitoKG-Grounded Mechanistic Inference

This folder provides the **LLM-based mechanistic interpretation** layer of
MitoLODE. It takes the candidate perturbation targets from
`perturbation/perturbation_forward.py` and produces two outputs:

1. A **step-by-step causal reasoning chain** for each target, grounded in
   MitoKG structural context + temporal dynamics + KGSFM gating weight,
   generated by a large language model (DeepSeek-Chat).
2. An **external validation score** for each target, computed from three
   databases independent of MitoKG.

| Script                          | Purpose                                                                 |
|---------------------------------|-------------------------------------------------------------------------|
| `mgmi_inference.py`             | Call the LLM to produce MitoKG-grounded reasoning chains.               |
| `mgmi_external_validation.py`   | Score candidates against OpenTargets + PubMed + Ensembl.                |

The two scripts form a pipeline; you must run them in this order.

## IMPORTANT: DeepSeek API key

`mgmi_inference.py` calls the DeepSeek Chat API. You must obtain your own
API key from <https://platform.deepseek.com> and expose it through the
environment variable `DEEPSEEK_API_KEY`. The script **refuses to run** if
the variable is not set. No API key is ever stored in this repository.

Linux / macOS:

```bash
export DEEPSEEK_API_KEY="sk-your-key-here"
```

Windows PowerShell:

```powershell
$env:DEEPSEEK_API_KEY="sk-your-key-here"
```

The external validation script only calls **public, unauthenticated** REST
APIs (OpenTargets GraphQL, NCBI PubMed eutils, Ensembl REST) and needs no
key.

## Prerequisites

- The MitoKG `gnn/` directory (`gene_embeddings.npy`, `gene_list.txt`,
  `node_id_maps.pkl`, `hetero_graph.pt`).
- `perturbation_lrrk2_temporal.xlsx` produced by
  `perturbation/perturbation_forward.py` (the `L2_GradientTargets`,
  `Summary`, `CrossTimepoint_GeneMatrix`, and `L1A_Top20_ByTimepoint`
  sheets are all used).
- `pip install openai openpyxl pandas numpy torch requests matplotlib`

## Usage

### Step 1 &mdash; LLM inference

```bash
export DEEPSEEK_API_KEY="sk-your-key-here"

python mgmi/mgmi_inference.py \
    --gnn-dir       ./MitoKG/gnn \
    --perturb-xlsx  ./output_perturbation_forward/perturbation_lrrk2_temporal.xlsx \
    --out-dir       ./output_mgmi \
    --mode          L2_top_N_each \
    --top-n-each    5
```

Three selection modes are available:

| `--mode`          | Behavior                                                           |
|-------------------|--------------------------------------------------------------------|
| `L2_top_N_each`   | Top-N gradient targets per time window (default 5 &times; 3 = 15). |
| `robust_top_N`    | Top-N cross-timepoint robust targets (ranked by max delta_r).      |
| `custom`          | User-supplied gene list via `--custom-genes`.                      |

Outputs (in `--out-dir`):

- **`mitokg_llm_raw_v2.json`** &mdash; one record per target with the full
  reasoning chain and automated verification flags. **Consumed by
  `mgmi_external_validation.py`.**
- `mitokg_llm_interpretability_lrrk2_v2.xlsx` &mdash; human-readable 2-sheet
  workbook (reasoning chains + verification summary).

Each reasoning chain is post-hoc verified:

- cited gene symbols must be present in MitoKG,
- temporal reasoning keywords (Day0 / Day10 / Day14 / window / plasticity)
  must appear,
- KGSFM gating keywords (embedding / norm / attention / gating) must appear,
- confidence must be `HIGH` or `MEDIUM`.

Chains satisfying all four conditions are flagged `PASS`.

### Step 2 &mdash; External validation

```bash
python mgmi/mgmi_external_validation.py \
    --mgmi-json ./output_mgmi/mitokg_llm_raw_v2.json \
    --out-dir   ./output_mgmi_validation \
    --top-n     15
```

The three external data sources are **disjoint from all five MitoKG
construction sources** (STRING, Reactome, DrugBank, CORUM, MitoCarta),
which eliminates the circular-validation concern:

| Source          | Role                                                      | Weight |
|-----------------|-----------------------------------------------------------|:------:|
| OpenTargets     | Parkinson's disease genetic association score (MONDO_0005180) | 0.50 |
| NCBI PubMed eutils | gene + "Parkinson" literature co-occurrence (log-normalized) | 0.30 |
| Ensembl REST    | gene biotype confirmation (protein_coding = 1.0, else 0.3)  | 0.20 |

Confidence tiers:

```
HIGH    : external_score >= 0.45 AND OpenTargets_score > 0
MEDIUM  : external_score >= 0.28
LOW     : otherwise
```

Outputs:

- `mgmi_external_validation.csv` &mdash; full record (all API response
  fields, scoring log, and metadata).
- `mgmi_external_validation_summary.csv` &mdash; compact comparison table.
- `fig_external_validation.png` &mdash; 4-panel figure (score per target /
  score decomposition / confidence level change / OpenTargets evidence
  breakdown).

## Complete pipeline

```
perturbation_forward.py
      |
      v
perturbation_lrrk2_temporal.xlsx   (L2_GradientTargets sheet)
      |
      |  +---> DEEPSEEK_API_KEY (env var)
      |  |
      v  v
mgmi_inference.py
      |
      v
mitokg_llm_raw_v2.json
      |
      v
mgmi_external_validation.py        (OpenTargets + PubMed + Ensembl)
      |
      v
mgmi_external_validation.csv       (+ summary + figure)
```

## Reproducibility notes

- LLM temperature is fixed at `0.3` and max tokens at `700`. Results may
  still vary slightly run-to-run because the API is non-deterministic.
  The reasoning chains reported in the paper were obtained with
  `deepseek-chat` on the model version served by the API during the
  experiment.
- OpenTargets and PubMed return live counts, so small numerical drift
  between runs is expected.
- OpenTargets uses **`MONDO_0005180`** for Parkinson's disease (the old
  `EFO_0002508` was deprecated). Several aliases are accepted by the
  filter to be robust against database migrations.
- A local Ensembl-ID cache is bundled for the 15 target genes most
  frequently produced by the perturbation module; for any other symbol
  the script falls back to an OpenTargets search query.
